jquery对象与dom对象

​ 学过js的不管会不会肯定都听过dom这个词，至少当时在上h5课的时候老师没少说。文档对象看的多了多少都是有概念的实际上的区别在于你是不是熟悉这些api，不常使用的话肯定也就没得说了。

如果知道dom操作的话就是通过getElementById()这种方式获得标签而jquery就是元素选择器的方式。

  1. 通过jQuery方法包装后的对象，是一个类数组对象。它与DOM对象完全不同，唯一相似的是它们都能操作DOM。
  2. 通过jQuery处理DOM的操作，可以让开发者更专注业务逻辑的开发，而不需要我们具体知道哪个DOM节点有那些方法，也不需要关心不同浏览器的兼容性问题，我们通过jQuery提供的API进行开发，代码也会更加精短。

dom元素与jquery之间的转换

​ 通常会在jquery对象前面加上$加以区分，打印出这个对象可以看到这是一个数组，下标为0的是这个dom元素，这就提供了对象到元素的转换方式，实际上就是通过拿到数组对象中的元素。后面的操作就转换成了对dom元素的操作。而dom元素想要转换成jquery对象则只需要将这个对象作为参数传给$()即可，这个元素会被包装成相对应的对象。

想要使用jQuery代码需要注意他的入口函数，也即任意代码应该包括在这个函数内部。

jQuery的特点有链式编程，实际上现在java里也有这种方式，在一个对象后面一直点点点。隐式迭代：隐式 对应的是 显式。隐式迭代的意思是：在方法的内部会为匹配到的所有元素进行循环遍历，执行相应的方法；而不用我们再进行循环，简化我们的操作，方便我们调用。

层级式的树状结构只有一个根目录，对于Linux来说一切皆为文件,系统中的硬件也会被映射成文件进行管理。

可以看到根目录的文件夹及文件。

/bin存放的内容为常用的命令 /home存放普通用户，每当你在系统中创建一个用户时，目录下就会对应生成这个用户的文件夹。 /root为用户权限的主目录。 /etc配置文件 /usr通常存放应用程序和相关文件。/dev 设备映射文件夹 /media识别u盘光盘等 /mnt通常为共享文件夹。/usr/local 安装软件的目录。

用户管理

众所周知，linux为多用户操作系统，对权限的管理十分重要，系统中可以有多个用户，不同的用户归于不同的组，不同的组拥有不同的权限，当用户属于那个组的时候也就拥有了对应的权限，所有的用户都在home目录下。

useradd 用户名创建用户后，同时也创建了一个同名的组，这个用户在这个组里，同时/home目录下多了一个文件夹。useradd -d /home/

用户名在指定文件夹下新建一个用户。passwd 用户名 给一个用户指定密码。 userdel 用户名 删除一个用户 userdel -r 用户名删除用户同时删除它的home目录。

id 用户名 查询用户的信息 如果用户不存在就会返回无此用户。

切换用户，权限不够的时候使用su命令进行切换。由高权限切换到低权限不需要输入密码，需要切回原来的用户主需要退出当前的用户即可exit。

jwt是什么

jwt全称Json Web Token。实际使用过程中可能对json也需要有一定的了解。但是在学完xml之后，对比与一系列的标签现在不过是又将他们规范为一些键值对而已。当然使用json的有点有很多，使用的资源少同时也更加容易理解，作为前后端数据交互的格式十分方便。

其基本结构为Header.Payload.Signature，本身是一个很长的字符串，三部分分别对应头部，负载，签名。头部存储的内容为算法和类型

负载中存储的内容可以使用官方提供的字段也可以定义私有字段

签名则是根据前两部分生成的，一旦前两部分的内容被修改，就会失效。

使用时需要首先指明一个加密的密钥，这个密钥只有服务器知道。经过加密之后将其产生的结果返回给用户。

用户在登录的时候除了一些必要的信息比如账号密码，还有可能会根据需要添加新的内容，把这些附加的信息单独取出来进行加密并在完成登录后返回给用户，以后用户在请求资源的时候只需要携带这串密文就可以验证是否有权限进行访问。同时它支持跨域访问。
