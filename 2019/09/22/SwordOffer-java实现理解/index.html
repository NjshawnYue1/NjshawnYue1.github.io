单例设计模式

实现单例模式的方法有很多。在学过的设计模式课程中提供的方式为提前加载，加同步锁等操作

或者在对外暴露的静态方法被执行时先判断本类对象是否为空以此实现单例，但在多线程条件下此方法仍然不安全。如果想要实现线程安全需要在对静态方法加synchronized 关键字修饰。

推荐的方式为使用静态内部类的方式，首先明白静态代码在类加载时只会被执行一次，在静态代码块中初始化对象，在对外暴露的方法中返回这个对象实现单例。

判断二维数组中是否包含目标元素

一个二维数组，每一行从左到右递增，每一列从上到下递增．输入一个二维数组和一个整数，判断数组中是否含有整数

双指针法

首先需要注意的是数组的特点，在规律上是有序的，查找算法中对有序序列使用二分查找效率应该是最高的。首先看双指针法，所谓双指针依然是根据二维数组本身已经有序才可以进行的，分别取出第一行第一列的元素与第一行的最后一个元素，这个序列本身应该是二维数组的一个元素(二维数组是存放数组的数组)，作为所谓的头尾指针元素去和目标值进行比较，这样可以通过比较结果来判断这个目标值应该在取出的这个一维数组中还是在剩余的数组中。

字符串查找替换

将一个字符串中的空格替换成”%20”

这题按照正常思路去对字符串进行遍历替换即可，在单线程条件下使用StringBuilder提高效率。逻辑比较简单，如果是空格则在对象中添加%20否则就把字符串中的值直接添加进对象中。

倒序打印链表

输入一个链表，从尾到头打印链表每个节点的值

除了要考虑如何倒序遍历之外，还是要清楚怎么在java中定义一个链表，那么在数据结构中，单链表将其中的节点分为两个部分，即数据域和指针域，通常为了便于操作，还会在第一个节点之前加一个头结点，可以通过头指针找到这个节点而后就可以对链表中的第一个节点进行操作。那么链表的优势就是可以在内存中物理逻辑上不连续的去存储一组数据方便数据的插入，但不利于随机访问。

那究竟该如何定义一个链表呢，对于上面的描述，链表类中应该包含一个数据以及链表本身的一个变量。通过访问next即可取到下一个节点。

完成定义后在类中需要通过实例化对象并手动将他们连接起来。对链表的遍历结合栈来操作实现倒序输出十分方便，需要进行的操作就变成了，只要链表不为空，则将这个节点入栈，然后访问这个节点的下一个节点，也就是访问这个节点的next属性。然后依次出栈即可。注意这里需要拿到的应该是节点中数据域里的内容。
