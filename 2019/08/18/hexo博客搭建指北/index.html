hexo博客搭建指北

Posted on 2019-08-18 Edited on 2020-10-21

HEXO 博客搭建

环境准备

安装node.js，访问官网安装最新版本直接安装即可。

打开终端 切换到管理员权限 sudo su

首先检查node是否安装成功，在终端输入node -v , npm -v 如出现相应版本则可以

安装淘宝npm镜像cnpm node$ npm install -g cnpm --registry=https://registry.npm.taobao.org 进行全局安装

安装完成后可以在终端使用cnpm命令

使用cnpm命令安装hexo node cnpm install -g hexo-cil 等待安装完成 使用hexo -v检查是否安装成

至此环境搭建就完成了

正式搭建

pwd 查看自己当前的路径 在根目录下使用mkdir hexo创建一个文件夹

cd hexo之后 使用命令 sudo hexo init等待执行完成这样初始化就完成了

可以看见当前文件夹下生成了很多个文件，那么需要使用的主要文件在source与themes文件夹下，以及最关键的配置文件_config.yml

使用命令 hexo s启动本地服务器，访问后可以看见相关内容，默认为你创建了一个hello world文件。

使用命令hexo new “你想要的文件名”进行创建，创建的文件为.md格式，这些步骤都是在hexo的根目录下完成的。

创建的文件在source目录下的_post文件夹里，可以看见刚刚自己新建的文件，使用编译器打开，我这里使用的是朋友推荐的一款Typora用来写md，使用md的基本语法进行书写，由于本人也是初学者所以不再献丑了

编辑完成后保存退出 回到hexo根目录下 使用命令hexo clean hexo g对目录先清理在重新生成，重新启动服务器hexo s访问本地服务器就可以看见刚刚自己创建的文件已经添加进来了

远端部署

访问自己的github地址，创建一个新的仓库。注意这里的命名格式是固定的切记不可更改，我在这里吃了亏浪费了很长时间所以单独提出来说一下，命名格式必须是自己的github用户名.github.io这个名字将来也是在浏览器中访问的地址，点击create创建

回到终端界面，想要部署到git上需要安装一个部署的插件，否则会提示不能识别Git这个命令，安装的方式为cnpm install --save hexo-deployer-git 等待安装完成即可，出现的waring可以不必在意

接下来的步骤也十分关键，上文提到的配置文件_config.yml使用文本编辑器将其打开，使用vim,sublime都可以，拉到最下面可以看到一个deployment标签在其下方进行相关配置。 deploy: type: git repo: 你的github项目地址 branch: master完成之后保存退出。

将博客部署到远端使用命令hexo d执行过程中需要输入你的github账号与密码，完成后即可

刷新自己的仓库可以发现文件以及被部署上来

访问自己的项目地址也就是上文提到的自己的github同户名.github.io 进行访问即可，访问时可能速度较慢需稍加等待，可进行刷新，成功后即可看见自己部署的内容。
